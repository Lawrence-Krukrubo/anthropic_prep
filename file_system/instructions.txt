Level 1: The Flat File System
As a reminder, you are building a basic flat file system (no directories yet, just files).

In your solution.py file, implement a FileSystem class that meets these exact specifications:

1. __init__(self): Initializes the file system.

2. create_file(self, name: str) -> bool: Creates a new empty file. 
Returns True if successfully created, False if a file with that name already exists.

3. write_to_file(self, name: str, content: str) -> None: Completely overwrites 
the file with the new content. If the file doesn't exist, it must raise a ValueError.

4. read_file(self, name: str) -> str: Returns the string content of the file. 
If the file doesn't exist, it must raise a ValueError.

5. delete_file(self, name: str) -> bool: Deletes the file. 
Returns True if successful, False if the file didn't exist.

Your Next Steps:

Level 2: Hierarchical File System
You must refactor your FileSystem class to support paths and folders. 
Every file and folder will now be referenced by an absolute path 
starting with / (e.g., /docs/resume.txt).

New Core Rules:

1. The root directory / always exists by default.
2. You cannot create a file or a directory if its parent directory does not exist.

Updated & New Methods:

1. create_directory(self, path: str) -> bool (NEW):
Creates a new directory. Returns True if successfully created. 
Returns False if it already exists OR if its parent directory doesn't exist 
(e.g., creating /a/b fails if /a doesn't exist).

2. create_file(self, path: str) -> bool (UPDATED):
Creates an empty file. Returns True if successful. Returns False if the file exists OR if its parent directory doesn't exist.

3. write_to_file(self, path: str, content: str) -> None (UPDATED):
Overwrites the file. Raises ValueError if the file doesn't exist or if path points to a directory.

4. read_file(self, path: str) -> str (UPDATED):
Returns file content. Raises ValueError if the file doesn't exist or if path points to a directory.

5. delete_file(self, path: str) -> bool (UPDATED):
Deletes the file. Returns True if successful, False if the file didn't exist.

(Hint: You will need to decide if you want to keep a flat dictionary 
where the keys are full string paths, or refactor into a 
nested dictionary / Node-based tree structure. 
Both are valid, but they have different tradeoffs!)
